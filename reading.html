Week 1

Programming Design System

There is a century-long bod between the field of design and new advances in technology, and if graphic designers do not become fluent in this new digital reality, they will become irrelevant. We now have the ability to write code that produces beautiful designs, and the designer of tomorrow will have to understand how to deliver on that promise. 
I really love what the author said in the preface, “There is a need for educational material that teaches the fundamentals of graphic design in a modern way, especially if it also gives students mental models for critiquing digital design projects.” With the emerging technology, the information could be mediated into different forms such as the website, digital phone and even augmented reality. The graphic design is mingled with other subjects like creative coding and interactive programming. The sophisticated system make it difficult to focus on one aspect like the graphic layout here. But the interface based on the aesthetic principle  is still important even when we do the generative art. 
Design systems are especially interesting today, because digital products are systems, and designers who code are no longer confined to the creation of design systems that end up in printed manuals. Code allows designers to not just create designs, but build digital systems that create designs.
It is interesting to see how the p5.js using code as a system to describe how to draw. Compared with others, the system of code is to set a series of drawing rules to create design. If we can call design as systems, the code can be also called a system of systems. Besides, the article especially in the chapter of “figure and ground”, I feel it pretty interesting to see the relationship between the shape and perception. I never think about how the graphic design can be trained in that way. By only playing with the size, place and rotation, you could decide the perception or feelings that you want to convey by your visual language.
